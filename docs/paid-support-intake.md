# Paid Support Intake

Use this when someone asks for optional paid setup help, workflow review, asset-library review, or sponsored Geometry Nodes work.

Keep basic alpha access, bug reports, and fixes for broken documented behavior free.

## Public Reply Template

```text
Thanks for trying NodeCue. Basic alpha access and bug reports stay free.

If you want paid help, I can look at setup support, a Geometry Nodes workflow review, a small private asset-library review, or a scoped sponsored fix.

Please do not post API keys, private asset-library paths, or unreleasable .blend files here. If this needs private files, we can move details to a private channel after confirming the scope.
```

## Private Intake Questions

Ask only what is needed to size the request:

- What do you want help with: setup, workflow review, asset-library review, or sponsored fix?
- What Blender version and OS are you using?
- Which provider/model are you using, if any?
- What is the smallest prompt or node graph that shows the issue?
- Can you share a screenshot, report JSON, or sanitized `.blend` file?
- What outcome would make the session useful?
- Is there anything private that should not be copied into public issues or docs?

## Starter Scope

Use small, low-risk scopes during alpha:

- Setup help: get NodeCue installed, configure sidecar Python, configure an OpenAI-compatible provider, and run one Generate prompt.
- Workflow review: review one Geometry Nodes task or generated graph and identify where NodeCue failed or helped.
- Asset-library review: inspect a small authorized node-group library and recommend indexing/reuse improvements without publishing private files.
- Sponsored fix: define one public reliability improvement and keep the resulting fix available in the public alpha.

Avoid broad production support promises while the add-on is still alpha.

## Boundaries

Do not accept paid work that requires:

- private model hosting;
- ChatGPT Plus or Claude Pro OAuth login support;
- production support guarantees;
- Shader Nodes or Compositing Nodes implementation in the first alpha;
- publishing private asset-library contents;
- hiding bug fixes that should remain public.

## After The Session

Capture only non-sensitive learnings:

- setup step that failed;
- model/provider issue;
- graph quality issue;
- explanation quality issue;
- asset reuse issue;
- docs or UI improvement needed.

If the lesson applies to public alpha users, open or update a public issue without private files, private paths, API keys, or customer-identifying details.
